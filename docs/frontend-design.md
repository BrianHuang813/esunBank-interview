# 前端設計規範

## 設計定位

介面定位為「編輯式館藏索引」。它需要具備金融服務的可信與克制，但不模仿銀行官網，也不套用制式管理後台。

設計參數：

- 版面變化：7 / 10
- 動態強度：2 / 10
- 資訊密度：5 / 10

## 色彩與材質

全站使用冷白背景、炭黑文字與深綠強調色。錯誤訊息可使用暗紅作為必要的語意色。

禁止使用：

- 漸層、外發光與玻璃效果
- box shadow 與 text shadow
- 多色卡片群
- emoji
- 只為裝飾存在的狀態圖示

層次由留白、字重、細線與位置建立。

## 排版與構圖

桌面頁面使用不對稱縮排、不等寬欄位與刻意保留的空白區域。書目採交錯索引，不使用固定三欄或四欄卡片。

不對稱用於頁面層級，表單欄位、館藏狀態與按鈕仍維持局部對齊，確保閱讀與鍵盤操作效率。小於 720px 時回到單欄或清楚的雙欄閱讀順序，禁止水平溢位。

頁面標題上限：

- 桌面：40px
- 手機：31px

## 頁面

### 館藏

首屏直接呈現搜尋與書目，不使用大型 Hero。書目使用交錯縮排，顯示作者、簡介、可借數與館藏數。

### 書籍明細

書封、書籍資訊與可借數採三段不等寬配置。館藏副本使用線性清單，不包成卡片。

### 登入與註冊

表單偏左，說明內容錯位放置於右側。標籤固定在欄位上方，錯誤固定在欄位下方，不以 placeholder 取代 label。

### 我的借閱

借閱中、已歸還與全部紀錄使用文字分頁。紀錄以流水帳呈現，歸還採原位置二次確認，不使用浮動 modal。

## 狀態與互動

每個資料頁需包含 loading、empty、error 與成功回饋。借還書按鈕在 request 期間 disabled，借閱衝突後重新取得館藏狀態。

Hover 只允許文字、底線、邊框與背景色改變，不使用浮起、縮放、傾斜、滑鼠追蹤或連續動畫。

## 書封

書封先呈現本地文字 fallback，再嘗試載入 Open Library ISBN 封面。遠端圖片成功後以灰階顯示；失敗或載入中都不會留下空白區塊。

## 驗收標準

- 桌面與 390px 手機皆無水平溢位。
- 頁面中不存在 CSS gradient、box shadow、text shadow 或 emoji。
- 表單支援鍵盤操作並具備明確 focus 樣式。
- 文字、輸入欄位與按鈕符合 WCAG AA 對比。
- 註冊、登入、借書、借閱紀錄與還書流程可由真實 API 完成。
